What is a List?

A list is a data structure that, using pointers, can link objects together into a single "chain".
It is very similar to an array with some major key differences:

  • Can't access an element by its index
  • Don't have to allocate a definite amount of memory at creation

Runtime Operations of a List

  • Traverse = O(n)

    • In order to go from the head of the list to its tail, every node along the list must be visited

  • Find

    • - Find is the base operation for everything else in linked list. All other runtime operations are basically fancy manipulations of find. Best case: O(1)
    - The desired node could be the head is instantaneous find
    Worst case: O(n)
    - The desired node could be the tail

  • Insert
    • Best case: Find position + constant operations = O(1) + O(1) = O(1)
    - The new node could be a value that should be the root. So the find will be instantaneous and the necessary manipulations are constant and so O(1)
    - OR, in an unordered list, items are added onto the beginning (as this O(1) runtime)
    Worst case: Find position + constant operations = O(n) + O(1) = O(n)
    - The new node could be a value that should be at the end of the list
  • Delete

    • Best case: Find position + constant operations = O(1) + O(1) = O(1)
    - The node to find could be the root. So the find will be instantaneous and the necessary manipulations are constant and so O(1)
    Worst case: Find position + constant operations = O(n) + O(1) = O(n)
    - The node to find could be the tail
